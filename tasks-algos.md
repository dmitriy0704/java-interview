# Задачи по алгоритмам

## Задача 1. «Поиск одинокого числа»

Условие:

Дан массив целых чисел. В этом массиве каждое число встречается дважды, и только одно число — один раз. 
Найдите это число.

Пример:

Вход: [2, 2, 1] → Выход: 1
Вход: [4, 1, 2, 1, 2] → Выход: 4

### Решение 1: Простое (через HashMap)
--------------------------------------

Мы можем пройтись по массиву и посчитать, сколько раз встречается каждое число.

```java

import java.util.HashMap;
public class Solution {
    public int singleNumber(int[] nums) {
        HashMap<Integer, Integer> counts = new HashMap<>();

        // Заполняем карту: число -> количество повторений
        for (int x : nums) {
            counts.put(x, counts.getOrDefault(x, 0) + 1);
        }

        // Ищем число, которое встретилось 1 раз
        for (int x : nums) {
            if (counts.get(x) == 1) {
                return x;
            }
        }
        return -1;
    }
}

```

Плюс: Легко читать.
Минус: Занимает дополнительную память (O(n)).


### Решение 2: Элегантное (через XOR)
--------------------------------------

В программировании есть побитовая операция XOR (исключающее ИЛИ), которая обозначается символом ^.
Свойства XOR:

   1. a ^ a = 0 (два одинаковых числа «обнуляют» друг друга).
   2. a ^ 0 = a (число при операции с нулем не меняется).
   3. Порядок не важен: (a ^ b ^ a) = (a ^ a ^ b) = 0 ^ b = b.

Если мы применим XOR ко всем числам в массиве, все пары «сгорят», и останется только то самое одинокое число.


```java
public class Solution {
    public int singleNumber(int[] nums) {
        int result = 0;
        for (int x : nums) {
            result ^= x; // Применяем XOR ко всем элементам
        }
        return result;
    }
}

```


Плюс: Работает максимально быстро (O(n)) и не требует лишней памяти.

### Почему это хорошая задача для практики?

   1. Она учит выбирать между понятным решением (HashMap) и эффективным (побитовые операции).
   2. Помогает понять, что иногда знание низкоуровневых операций (как XOR) может сильно упростить код.

## Задача 2. «Является ли строка палиндромом?»

Для практики работы со строками в Java отлично подходит задача на проверку палиндрома. 
Она учит работать с индексами, методами класса String и пониманию того, как эффективно обходить данные.

Условие:

Напишите метод, который проверяет, читается ли строка одинаково в обоих направлениях. 
При этом нужно игнорировать регистр (большая или маленькая буква) и все символы, кроме букв и цифр.

Примеры:

* "A man, a plan, a canal: Panama" → true
* "race a car" → false


### Решение: Метод двух указателей

Самый эффективный способ — идти с двух сторон строки одновременно (начало и конец) к центру.

```java

public class PalindromeChecker {
    public boolean isPalindrome(String s) {
        // Указатели на начало и конец строки
        int left = 0;
        int right = s.length() - 1;

        while (left < right) {
            char leftChar = s.charAt(left);
            char rightChar = s.charAt(right);

            // 1. Пропускаем символы слева, если это не буква или цифра
            if (!Character.isLetterOrDigit(leftChar)) {
                left++;
            } 
            // 2. Пропускаем символы справа, если это не буква или цифра
            else if (!Character.isLetterOrDigit(rightChar)) {
                right--;
            } 
            // 3. Сравниваем символы в нижнем регистре
            else {
                if (Character.toLowerCase(leftChar) != Character.toLowerCase(rightChar)) {
                    return false; // Не совпало — не палиндром
                }
                left++;
                right--;
            }
        }
        return true;
    }

    public static void main(String[] args) {
        PalindromeChecker checker = new PalindromeChecker();
        System.out.println(checker.isPalindrome("Madam, I'm Adam")); // true
    }
}

```

### Разбор ключевых моментов

   1. Два указателя (left и right): Мы не создаем новую перевернутую строку (что сэкономит память), а просто сравниваем символы «на месте».
   2. Класс Character: Методы isLetterOrDigit() и toLowerCase() — это стандартные инструменты Java для фильтрации и нормализации текста.
   3. Эффективность:
   * Сложность по времени: $O(n)$, так как мы проходим по строке всего один раз.
      * Сложность по памяти: $O(1)$, мы используем только две переменные для индексов.
   
### ✅ Ответ
Задача решается за линейное время с помощью двух указателей, что позволяет избежать создания лишних объектов в памяти.

##  Задача 3: "Поиск суммы двух чисел (Two Sum)".

### Условие

Дан массив целых чисел nums и целое число target. Нужно вернуть индексы двух чисел так, 
чтобы их сумма была равна target.

* Предполагается, что каждое входное значение имеет ровно одно решение.
* Нельзя использовать один и тот же элемент дважды.
* Результат можно вернуть в любом порядке.

### Решение 1: Метод "Брутфорс" (Наивное)
Мы просто перебираем все возможные пары чисел с помощью двух вложенных циклов.

* Сложность по времени: $O(n^2)$
* Сложность по памяти: $O(1)$

```java
public int[] twoSum(int[] nums, int target) {
    for (int i = 0; i < nums.size(); i++) {
        for (int j = i + 1; j < nums.size(); j++) {
            if (nums[i] + nums[j] == target) {
                return new int[] { i, j };
            }
        }
    }
    return new int[] {};
}
```


### Решение 2: Оптимальное (с использованием HashMap)

Мы проходим по массиву один раз. Для каждого числа x мы проверяем, есть ли в таблице 
число target - x. Если есть — мы нашли пару. Если нет — сохраняем текущее число и его индекс 
в таблицу.

* Сложность по времени: $O(n)$
* Сложность по памяти: $O(n)$

```java

import java.util.HashMap;import java.util.Map;
class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            
            // Если мы уже видели "дополнение" к текущему числу
            if (map.containsKey(complement)) {
                return new int[] { map.get(complement), i };
            }
            
            // Иначе сохраняем число и его индекс
            map.put(nums[i], i);
        }
        
        throw new IllegalArgumentException("Решение не найдено");
    }
}
```

## Задача 4: «Валидация скобок»

Стек работает по принципу LIFO (Last In, First Out) — «последним пришел, первым ушел». 
Представь стопку тарелок: ты кладешь новую сверху и снять можешь только верхнюю.
Классическая задача на эту тему — «Правильные скобки».

Условие:

Дана строка, состоящая только из символов (, ), {, }, [ и ]. Нужно определить, является ли такая строка «правильной».

Строка правильная, если:

   1. Каждая открывающая скобка закрывается скобкой того же типа.
   2. Скобки закрываются в правильном порядке.

Примеры:

* "()[]{}" → true
* "(]" → false
* "([)]" → false (нарушен порядок)
* "{[]}" → true


### Решение через Stack

В Java для этого удобнее всего использовать класс Deque (например, ArrayDeque), 
так как старый класс Stack считается устаревшим.

```java

import java.util.ArrayDeque;import java.util.Deque;
public class Solution {
    public boolean isValid(String s) {
        Deque<Character> stack = new ArrayDeque<>();

        for (char c : s.toCharArray()) {
            // Если скобка открывающая — кладем её в стек
            if (c == '(' || c == '[' || c == '{') {
                stack.push(c);
            } 
            // Если закрывающая — проверяем верхушку стека
            else {
                // Если стек пуст, а пришла закрывающая скобка — строка неверна
                if (stack.isEmpty()) return false;
                
                char top = stack.pop(); // Достаем верхний элемент
                
                // Проверяем соответствие типов
                if (c == ')' && top != '(') return false;
                if (c == ']' && top != '[') return false;
                if (c == '}' && top != '{') return false;
            }
        }

        // В конце стек должен быть пустым
        return stack.isEmpty();
    }
}

```

------------------------------
## Почему это важно понимать?

   1. Принцип работы: Стек идеально подходит для задач, где нужно «запоминать» 
   состояние и возвращаться к нему (как кнопка «Назад» в браузере или Ctrl+Z).
   2. LIFO в действии: Когда мы видим ( [ ] ), мы сначала кладем ( в стек, потом [. 
   Когда приходит ], мы сравниваем её именно с последней положенной скобкой — [. Они совпали, 
   мы их «аннигилировали» и идем дальше.
   3. Память и время: Это решение работает за $O(n)$ по времени и $O(n)$ по памяти в худшем 
   случае (если все скобки открывающие).
